04 — The Path Forward

What Is Needed

To get from today’s fragmented landscape to an agent-ready design system, these are the six foundational moves we need to make.

1

Redo CSS Architecture

Rebuild our styling layer around semantic aliases and component-level hooks — with fallback states baked in so any surface can be branded without rewriting CSS.

  • Semantic token aliases (var(--color-on-surface-1)) replacing raw hex
  • Component styling hooks for overrides without breaking internals
  • Fallback state tokens for graceful degradation across brands
2

Create Composable Components

Templatise blades and cards into composable building blocks. Fewer primitives, infinite combinations — reduce catalog size and ambiguity.

Before 25+ blades · 15+ cards
After ~5 composable templates
3

Interactive Storybook & Documentation

Transform Storybook from a static visual catalog into an interactive, API-driven platform that serves both humans and machines.

  • Interaction tests that LLMs can invoke to validate rendered output
  • Prop/slot discovery APIs for agents to query component capabilities
  • Visual regression wired into agent feedback loops for self-correction
  • Vendor-facing docs enabling partners to build without hand-holding
4

Unify Into a Single Library

Consolidate scattered component sets into one React-based library — a single bundle that every surface imports from.

Goal One library, one tech stack, one source of truth across all MFEs
5

Build LLM Routing Logic

Create the machine-readable contracts — schema, recipes, constraints — that let LLMs route prompts to the right components and assemble pages autonomously.

  • Intent-to-recipe mapping for adaptive website generation
  • Token & constraint validation before render
  • Pluggable agent architecture for content, media, SEO, and beyond
6

Agent-Powered Migration

Use the agents and routing logic we’ve built to accelerate migration of existing surfaces — turning a multi-quarter effort into weeks.

Outcome Agents scan legacy pages, map to composable templates, and generate migration PRs automatically
